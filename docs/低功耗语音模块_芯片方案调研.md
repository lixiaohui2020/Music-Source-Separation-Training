> 调研日期：2026-07  
> 用途：为自研 Audio Core / always-on 语音子系统提供行业对标与架构参考  
> 说明：公开资料汇总，部分功耗数字来自厂商宣传/白皮书，立项前需自行实测或索取 NDA 资料

## 1. 行业共识架构：分级唤醒链

几乎所有成熟方案都采用**多级电源域 + 分级唤醒**，而不是单一大核常开：

```
Mic (PDM/I2S)
   ↓
① 能量/活动检测（模拟或极简数字，μW～百 μW）
   ↓
② VAD / 轻量前端（百 μW～1 mW）
   ↓
③ KWS / 命令词（1～数 mW，小 DSP / 小 NPU）
   ↓
④ 命中后上电：大 DSP / NPU / AP（做 AEC、降噪、ASR、LLM…）
```

对自研 Audio Core 的启示：

- **always-on 必须独立电源域**（retention SRAM + 慢时钟）
- KWS 层应**可编程/可换权重**（否则唤醒词锁死）
- 大算力（Transformer / 重降噪）留在主 NPU，不进 always-on 域

## 2. 方案分类一览

| 类型 | 代表 | 典型形态 | 适合对标场景 |
|---|---|---|---|
| **SoC 内置 always-on 语音子系统** | Qualcomm、Ambiq、恒玄 BES | 主 SoC 内嵌低功耗音频岛 | 手机/可穿戴/IoT 一体芯片 |
| **可授权音频 DSP IP** | Cadence HiFi、CEVA BX | 买 IP 集成进自研 SoC | 自研芯片最直接参考 |
| **外挂低功耗语音加速器** | Knowles IA8201、Syntiant NDP | 独立小芯片挂主控旁 | 主芯片无音频岛时的补丁方案 |
| **语音专用 MCU/AI MCU** | Synaptics SR80、乐鑫 ESP32-S3 | 单芯片完成唤醒+音频 AI | 音箱/耳机/门锁等端侧产品 |
| **AIoT Wi-Fi 音视频 SoC** | 博通集成 BK7258 | 连接 + 音频 + 轻 AI | 门锁/IPC/面板 |

## 3. 厂商方案详解（附链接）

### 3.1 Qualcomm —— LPASS + Hexagon 多级 Listen

**架构要点**

- **LPASS（Low-Power Audio Subsystem）** 独立音频子系统  
- 典型链路：Codec 侧 **MAD（Mic Activity Detection）** → Hexagon **WDSP/ADSP** 跑 Listen/KWS → 再唤醒 APSS  
- Snapdragon Voice Activation（SVA）支持 OEM/用户自定义唤醒短语  
- 音频与传感可分核：aDSP（音频）、Sensing Hub DSP（always-on 传感）

**对自研启示**

- “Codec 硬件活动检测 + DSP KWS + AP 重任务”是业界标杆分层  
- 唤醒词不在硬化 ROM，而在 DSP 上的可更新模型  

**链接**

- Hexagon 架构论文：https://pages.cs.wisc.edu/~danav/pubs/qcom/hexagon_micro2014_v6.pdf  
- Hexagon 简介：https://en.wikipedia.org/wiki/Qualcomm_Hexagon  
- Lantronix 对 SVA/Listen 多级检测说明：https://www.lantronix.com/blog/leverage-qualcomms-socs-voice-control-capabilities-system-modules-lantronix/  
- QRB5165 特性（含 LPASS / Sensing Hub）：https://docs.qualcomm.com/doc/80-PV086-5P/topic/QRB5165-features.html

### 3.2 Ambiq —— SPOT 超低功耗 + always-on 音频前端

**架构要点**

- Apollo510：Cortex-M55 + Helium，强调**多数 AI 可不依赖独立 NPU**  
- 专用低功耗音频：LP Audio ADC、PDM DMIC、I2S(+ASRC)  
- 官方明确支持 **ultra-low power voice / keyword detect**  
- soundKIT：开源音频 AI SDK（KWS、VAD、增强、声纹），heliaRT 加速  

**对自研启示**

- always-on 音频外设（ADC/PDM）与推理核同岛，是 sub-mW 唤醒的前提  
- 软件栈（KWS/VAD 模型 + 运行时）与硬件同等重要

### 3.3 Cadence Tensilica —— HiFi 1s / HiFi 5“大小核”音频 IP

**架构要点**

- **HiFi 1 / HiFi 1s**：专为 always-on、TWS、蓝牙编解码、KWS 设计的超低功耗 DSP  
- 带 NN ISA / AI-MAC，可跑 wake-word（白皮书给过 “OK Google” TFLM 能效对比）  
- **Big-Little**：HiFi 1 always-on 听唤醒 → 唤醒 HiFi 5 做重音频/更大词表  
- HiFi 5/5s：高性能音频 + 扩展 NN；可再挂 NNE 神经网络加速器  

**对自研启示**

- 最接近“路线 D（AFE 硬化 + 可编程小核）”的商业 IP 形态  
- **建议重点对标**：若自研人力不足，HiFi 1s + FFT/FIR 硬加速是务实选项

### 3.4 CEVA —— BX1 / BX2 + WhisPro 语音引擎

**架构要点**

- **CEVA-BX1**：面向 TWS / always-on 的超低功耗音频 DSP（兼控制）  
- **CEVA-BX2**：更重的波束、ASR、NLP 等  
- **WhisPro**：always-on 唤醒词 + 命令词软件，支持自定义唤醒词；可基于 TFLM 部署  
- ClearVox：多麦降噪 / AEC  

**对自研启示**

- “可编程 DSP IP + 可定制 KWS 软件包”是完整交付形态  
- 自定义唤醒词是标配能力，不是例外

### 3.5 Synaptics —— AudioSmart 远场 + Astra SR80 Audio AI MCU

**架构要点**

- 经典方案 **CX20921**：独立双麦远场 SoC，内置低功耗 **Wake-on-Voice**，WoV 模式宣称 <70 mW（面向家电待机法规，非可穿戴 μW 级）  
- 新一代 **Astra SR80**：Cortex-M33 + **HiFi 5** + 自研低功耗 DSP（CAPE2）+ **NPU** + Codec  
  - 低功耗 DSP 子系统做 ANC/低时延；NPU 做音频 AI  
  - 典型“多引擎分工”：小 DSP / NPU / 大 HiFi 各管一段  

**对自研启示**

- 外挂语音前端 SoC 可快速补齐主芯片短板  
- 新一代明确走向 **DSP + NPU 异构**，与“Audio Core + 图像 NPU”思路同构  

**链接**

- CX20921 PDF：https://www.synaptics.com/sites/default/files/audiosmart-dual-mic-far-field-voice-cx20921.pdf

### 3.6 Knowles AISonic IA8201 —— 双核 OpenDSP 外挂语音处理器

**架构要点**

- **双核**：HemiDelta（低功耗 always-on）+ DeltaMax（高算力 AFE/ML）  
- 最高 4×PDM mic，1MB 用户 RAM（可存多算法/多关键词）  
- 支持 on-device keyword trigger、二级验证、再上云 ASR  
- 带 FFT / DNN 等音频加速指令  

**对自研启示**

- **大小核同芯片**是语音加速器经典结构（小核常听，大核做波束/AEC/校验）  
- 大 SRAM 存多唤醒词模型 → 换词不改硅  

**链接**

- IA8201 Product Brief：https://www.knowles.com/docs/default-source/default-document-library/knowles-ia8201-product-brief-final9d761b731dff6ddbb37cff0000940c19.pdf

### 3.7 Syntiant NDP120 —— 专用 Neural Decision Processor

**架构要点**

- 专为 always-on 语音/传感推理设计的 **Neural Decision Processor**  
- Syntiant Core 2：原生跑 CNN/RNN/LSTM/GRU 等，高能效张量吞吐  
- 内嵌 **HiFi 3 DSP**（经典音频处理）+ Cortex-M0（管理）  
- 多唤醒词、本地命令、事件分类、AEC/NS/波束等并发网络  
- PDM / I2S/TDM 直连  

**对自研启示**

- “小 NPU 专做 always-on 语音”可做到极低功耗；与图像大 NPU 彻底解耦  
- 若自研 Audio Core 的 NN 执行单元，可参考其**支持的层类型清单**（含 LSTM/GRU）  

**链接**

- NDP120 官网：https://www.syntiant.com/ndp120/

### 3.8 恒玄 Bestechnic（BES）—— TWS Sensor Hub + BECO NPU

**架构要点**

- BES2700 系列：TWS / ANC / AI Voice 超低功耗蓝牙音频 SoC  
- 典型异构：  
  - 应用 CPU（如 Cortex-M55）± HiFi 4  
  - **Sensor Hub**：STAR-MC1 + **BECO NPU** + 自研信号/NN 协处理  
- Sensor Hub 承担低功耗传感与 AI 推理（含语音相关负载），主核可睡  

**对自研启示**

- 国内 TWS 龙头的标准答案：**独立 Sensor Hub + 小 NPU**，不是把 KWS 塞进主核  
- 与报告推荐的“Audio Core always-on 岛”高度同构

### 3.9 乐鑫 Espressif —— ESP-SR / WakeNet（MCU 指令扩展路线）

**架构要点**

- ESP32-S3：Xtensa + **AI/DSP 指令扩展**，无独立大 NPU  
- **ESP-SR**：WakeNet（唤醒）+ MultiNet（命令词）+ AFE（AEC/VAD/NS/BSS）  
- 支持多语言预设词，并提供**自定义唤醒词**训练/部署路径  
- 成本与生态友好，功耗与效果介于“专用语音芯片”和“纯 MCU”之间  

**对自研启示**

- 证明：**可编程小模型 + 软件栈**即可交付可用唤醒，不一定先做硬引擎  
- 自定义唤醒词流程可作为算法团队工具链参考  

**链接**

- ESP-SR（含 WakeNet）：https://github.com/espressif/esp-sr  
- WakeNet 文档：https://docs.espressif.com/projects/esp-sr/en/latest/esp32s3/wake_word_engine/index.html

### 3.10 博通集成 Beken BK7258 —— Wi-Fi 音视频低功耗 SoC

**架构要点**

- Wi-Fi 6 + BLE 5.4 + STAR-MC1（最高 480 MHz，DSP/SIMD）  
- 音频外设：双 Audio ADC、DAC、DMIC；软件侧有 Audio Engine（AEC/NS/编解码）

**对自研启示**

## 5. 对自研 Audio Core 的直接建议（基于本调研）

1. **唤醒链采用业界标准三级**：活动检测 → VAD/轻前端 → KWS → 再上电 NPU；不要让图像 NPU 常开听麦。  
2. **KWS 不要全硬化 ROM**：权重放 SRAM/Flash，支持 OTA 换词；与 Qualcomm/CEVA/ESP/Syntiant 一致。  
3. **硬件边界对齐 Cadence/Knowles/BES**：  
   - 硬化：FFT/FIR/mel/窗/OLA/LUT（AFE）  
   - 可编程：Conv1D / 小 RNN / KWS 调度（小 DSP 或微码 + 小 MAC）  
4. **优先找两家做深度对标**（建议）：  
   - **Cadence HiFi 1s**（IP 可行性、面积/功耗量级）  
   - **恒玄 BES2700 Sensor Hub**（国内量产形态、Sensor Hub + NPU 切分）  
5. **软件栈必须立项同步**：WakeNet/WhisPro/soundKIT 说明——没有 KWS 工具链，硬件岛用不起来。
